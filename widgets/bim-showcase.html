Hong Kong Commercial Project
Multi-discipline clash coordination · 2024 · Client confidential
Type
Commercial
Disciplines
ARC · STR · MEP
Clash tests
4 tests
Software
Navisworks
Federated BIM model across architecture, structural, and MEP disciplines. Ran four clash detection tests in Navisworks to identify and resolve coordination conflicts before construction. Coordinated clash resolution with the design team across multiple revision cycles.
Four clash tests executed
  • T01: ARC vs STR — architecture against structural framing
  • T02: ARC vs MEP — architecture against MEP routing
  • T03: STR vs MEP — structural against MEP services
  • T04: MEP vs MEP — internal MEP coordination
VDC workflow applied
  • Discipline model export (Revit → NWC)
  • Model federation in Navisworks Manage
  • Clash detective rule setup (hard, clearance, duplicate)
  • Issue documentation with saved viewpoints
  • Coordination markup and reissue cycle
Why this proof point matters
Same federated coordination workflow used on US healthcare and commercial mega-projects. International experience with multi-discipline BIM at production scale.

Case Study 03
BrightSource Solar Project — Risk Analysis
Monte Carlo simulation · PJM 6015 · Northeastern · March 2025
Tool
Primavera RA
Iterations
10,000
Tasks modeled
53
Distribution
Triangle
Ran a 10,000-iteration Monte Carlo simulation in Primavera Risk Analysis across a 53-task construction project. Modeled schedule and cost uncertainty using Triangle distributions. Identified top sensitivity drivers and recommended a data-backed completion date and contingency budget.
Key findings delivered
  • 80% confidence completion: Jan 27, 2026
  • Recommended budget: $835K with $850K contingency cap
  • Top schedule risk driver: Monitor & Control Project Work (165% SSI)
  • Top cost risk driver: Financing/Contracts (61% sensitivity)
  • Critical foundational tasks: Project Charter, Kick-off Meeting, High-Level Planning (100% criticality index)
Analyses produced
  • Project finish distribution histogram
  • Project cost distribution curve
  • Schedule sensitivity tornado graph
  • Cost sensitivity tornado graph
  • Schedule Sensitivity Index (SSI) tornado
  • Criticality Index tornado
  • Duration Cruciality analysis
Why this proof point matters
Modern GCs run quantitative risk on every major project. PRA + Monte Carlo is the industry standard. Most co-op-stage candidates have never built one — I have.
